# Assembly Comparison Tests in C, Haskell, and Python

This project aims to compare the assembly code generated by different programming languages (C, Haskell, and Python) for algorithms that analyze complexity growth in terms of operations, time, and memory.

## Naming Convention

```markdown
<testType><number>_<language>.<extension>
e.g.: ops1_C.c, time2_Hs.hs, mem3_Py.py
```

### Script PowerShell to Compile Haskell

Use:

```pwsh
.\compile.ps1 <FilePath\FileName>
```

Example:

```pwsh
 .\compile_hs.ps1 path/filename.hs
 .\compile_hs.ps1 .\growth\minimal\min_HS\min5_HS\min5_HS.hs
```

Or use the following commands directly in the terminal:

```sh
# Generate assembly code with different optimization levels
gcc -S ./filename.c -o filename.s
gcc -S -O1 ./filename.c -o o1_filename.s
gcc -S -O2 ./filename.c -o o2_filename.s
gcc -S -O3 ./filename.c -o o3_filename.s

# Compile the executable
gcc ./path/filename.c -o ./path/filename.exe
```

### Script to Generate Assembly in Haskell

```sh
# Generate assembly code with different optimization levels
ghc -S -o filename.s filename.hs
ghc -S -O1 -o o1filename.s filename.hs
ghc -S -O2 -o o2filename.s filename.hs
ghc -S -O3 -o o3filename.s filename.hs

# Compile the executable
ghc ./path/filename.hs -o ./path/filename.exe
```
